# Phase 3 — UI Design Contract

> Visual and interaction contract for Phase 3: Releases.
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.
>
> **Phase goal:** A visitor browses the full Marginalia catalog as an artwork-led grid
> and lands on release detail pages with metadata, a client-only SoundCloud embed, and
> buy/stream links. Release pages are shareable with correct OG previews and expose
> `MusicAlbum` JSON-LD.
>
> **Brand context:** Marginalia — melodic house & techno label, Barcelona. Dark-first,
> high-contrast, underground aesthetic. Release artwork is the hero on every surface.
> The shell recedes; the covers do the talking.

## Typography

Inherited verbatim from Phase 2. Nimbus Sans (Regular 400 + Bold 700). Two weights
only, no italics. Font sizes declared as four `@theme {}` tokens.

| Role | Size | Weight | Line Height | CSS var | Phase 3 Usage |
|------|------|--------|-------------|---------|---------------|
| Body | 16px | 400 | 1.5 | `--text-body` | Description prose, "More platforms" summary, general copy |
| Label | 12px | 400 | 1.4 | `--text-label` | Catalog number, genre chip text, release date meta, hover overlay text |
| Heading | 24px (32px desktop) | 700 | 1.2 | `--text-heading` | Release title on detail page, catalog page H1, "More platforms" section heading |
| Display | 48px (64px desktop) | 700 | 1.1 | `--text-display` | Not used in Phase 3 — reserved for homepage hero (Phase 5) |

**Phase 3 typography rules:**
- Release title on detail page uses `--text-heading` (24px mobile / 32px desktop), bold,
  letter-spacing `--tracking-tight` (-0.02em).
- Artist name on detail page: `--text-body` (16px), Regular 400, rendered as a link to
  `/artists/[slug]` (404 until Phase 4 — acceptable per D-10).
- Release date and catalog number: `--text-label` (12px), Regular 400, color
  `--color-text-secondary`.
- Genre chips (if rendered in secondary section): `--text-label` (12px), Regular 400,
  background `--color-tag-*` (see Phase 2 tag palette), text `--color-bg` (#1F1F21) for
  contrast on lighter tag fills; `--color-text-primary` for darker tag fills.
- Catalog grid cards are **artwork-only, no text on card surface** (D-01). The hover
  overlay text is 12px Label / 16px Body pair — see Interaction States.
- Laylo CTA button label: `--text-body` (16px), weight 700 — the only UI element in
  this phase that uses body size at bold weight, to make the accent CTA scannable.
- "More platforms" disclosure summary: `--text-label` (12px) uppercase, letter-spacing
  +0.08em, color `--color-text-secondary`.

No new typography tokens introduced in Phase 3.

### Accent assignment (10% — resolved from D-16)

Decision: **Laylo CTA default fill = `--color-accent-lime` (#9EFF0A), hover fill =
`--color-accent-violet` (#580AFF).** Lime is the "energy / action" cue across the site
(already used for focus rings and active nav underline in Phase 2). Violet is reserved
for the press state and for primary buttons on future forms (Phase 6).

**Accent reserved for, on pages in this phase:**
- Laylo presave CTA button fill (lime default, violet on hover, lime-dark text `--color-bg` for contrast)
- Keyboard focus ring on all interactive elements (lime, 2px, offset 3px — inherited rule)
- SoundCloud player accent (lime, passed as iframe color param for visual mode only)

**NOT accent** on these pages:
- Artist links on the detail page — these render as `--color-text-primary` with
  underline on hover, not accent color
- Platform icon row (Beatport / Spotify / SoundCloud / Apple Music / Deezer / Bandcamp)
  — icons use `--color-text-secondary` default, `--color-accent-lime` on hover
  (inherited `SocialIcon` pattern from Phase 2)
- Genre chips — these use the `--color-tag-*` palette (11 colors), NOT the 10% accent.
  Chips are decorative meta, not actions.
- Hover overlay on catalog cards — the overlay is `--color-bg` at 70% opacity; the
  revealed text is `--color-text-primary`. No accent on card surface.

### Genre chip color mapping

Chips are rendered in the secondary metadata section of the detail page (Claude's
discretion from D-09 — resolved: **yes, render in secondary section below the embed,
above "More platforms"**).

| Genre value | Chip token | Background | Text color |
|-------------|-----------|------------|------------|
| melodic-house | `--color-tag-lavender` | `#b088d0` | `--color-bg` (#1F1F21) |
| techno | `--color-tag-blue` | `#2086c0` | `--color-text-primary` (#FFFFFF) |
| indie-dance | `--color-tag-pink` | `#ef6b8e` | `--color-bg` (#1F1F21) |
| organic-house | `--color-tag-mint` | `#66cc99` | `--color-bg` (#1F1F21) |
| afro-house | `--color-tag-orange` | `#f29753` | `--color-bg` (#1F1F21) |

Contrast rule: chip text is `--color-bg` on any tag whose background is lighter than

any additional genre added post-v1 must be assigned a chip token at schema-edit time.

### Hover overlay on catalog cards (resolves D-03 Claude's discretion)

- Background: `--color-bg` at 70% opacity (`rgba(31,31,33,0.70)`) — solid, not gradient.
  Solid chosen because gradient implies a direction and competes with artwork
  composition; solid keeps the card quiet.
- Transition: 200ms ease-out on opacity. Initial opacity 0 (desktop hover only).
- Content position: centered, stacked. Title on top (Body 16px, weight 700), artist
  name below (Label 12px, weight 400, `--color-text-secondary`), 8px gap.
- Mobile/touch: overlay is suppressed entirely (D-04). Card is purely clickable artwork.

## Layout Specification

### `/releases` — Catalog Grid Page

**Structure:**
- Page top padding: 64px (`--space-3xl`) on desktop, 48px (`--space-2xl`) on mobile.
- Page H1: "Releases" — `--text-heading`, bold, `--color-text-primary`, margin-bottom
  32px (`--space-xl`).
- Grid: 3/4/5 columns (mobile/tablet/desktop), 16/20/24px gap.
- Each card: square 1:1 aspect ratio, `object-cover`, cover art fills card (D-05).
- Page bottom padding: 64px (`--space-3xl`) on desktop, 48px (`--space-2xl`) on mobile.

**Card behavior:**
- Wrapped in `<a href="/releases/{slug}">` — whole card clickable.
- Desktop hover: 200ms fade of dark overlay (rgba(31,31,33,0.70)) with title + artist
  name centered on top. (D-03)
- Mobile/touch: no overlay. Tap navigates directly. (D-04)
- Focus-visible: 2px lime outline at 3px offset, on the card link (accessibility parity
  with hover).
- Keyboard Enter/Space on focused card: navigates.

**Sort:** release date descending (newest first) (D-06). No filter controls in Phase 3.

**Empty state:** if `reader.collections.releases.all()` returns 0 entries, render the
empty-state block (see Copywriting Contract). This is a content-management fallback,
not expected in production.

### `/releases/[slug]` — Release Detail Page

**Mobile layout (< 1024px), top-to-bottom:**
1. Cover art — full-width, 1:1 aspect ratio, `next/image` with `priority` + explicit `sizes="100vw"` (Claude's discretion from D-22 resolved: **use `next/image` with `sizes`**, not `fill`, because fixed-aspect 1:1 is compatible with responsive image generation).
2. Metadata header block: title (Heading), artists (Body), release date (Label). 16px vertical rhythm between lines, 24px gap to next block.
3. Laylo CTA button (if `layloUrl` present). Full-width on mobile, lime fill.
4. Primary platform icon row: Beatport, Spotify, SoundCloud, Apple Music, Deezer, Bandcamp — rendered horizontally centered, 16px gap, icons only (reuses `SocialIcon` visual pattern). Renders nothing if all primary URLs are empty.
5. SoundCloud embed (if `soundcloudUrl` present) — skeleton placeholder while hydrating, then 166px-tall iframe in visual mode. If empty, the section is omitted entirely (D-13).
6. Description — `DocumentRenderer` output (prose), body size, 1.5 line height.
7. Secondary metadata section: catalog number, release type, genre chips. Rendered below description, separated by 48px (`--space-2xl`).
8. "More platforms" collapsible (if any secondary URL present). Default collapsed.
9. Page bottom padding: 64px.

**Desktop layout (≥ 1024px), two-column:**
- **Left column** (sticky, ~40% of container): cover art 1:1, sticks at top with 32px
  offset from nav-height (`calc(var(--nav-height-desktop) + 32px)`). Cover art fills
  column width.
- **Right column** (~60%, scrolls): metadata header → Laylo CTA → platform icon row →
  SoundCloud embed → description → secondary metadata → "More platforms."
- Column gap: 48px (`--space-2xl`).

**Sticky artwork rule:** left column uses `position: sticky; top: calc(var(--nav-height-desktop) + 32px);`. The right column is the scroll driver.

**Artist link rule:** on the detail page, artist names are rendered as
`<a href="/artists/{slug}">` regardless of whether that page exists yet (D-10). Until
Phase 4 ships, these 404. Do not add `aria-disabled` — these are real forward links,
not placeholders.

## SoundCloud Embed Contract

**URL construction (D-12, resolved):**
```
https://w.soundcloud.com/player/?url={encodeURIComponent(soundcloudUrl)}
  &color=%239EFF0A
  &auto_play=false
  &hide_related=true
  &show_comments=false
  &show_user=true
  &show_reposts=false
  &show_teaser=false
  &visual=true
```

Lime accent (`%239EFF0A` = URL-encoded `#9EFF0A`) as player color — ties to the 10%
accent contract. `visual=true` gives the artwork-forward display that reads best on
dark background.

**Height:** 166px (SoundCloud standard for visual mode). Fixed, no responsive scaling.

**Skeleton placeholder:** 166px tall block, `--color-surface` (#2A2A2C) background,
no animation, no text. Renders only until the client-side dynamic import resolves.
Subtle, not a "loading..." spinner. (Claude's discretion from D-22 — resolved: **no
animation** to keep the page quiet and prevent perceived flicker.)

**Null handling:** if `soundcloudUrl` is empty/null, the embed section including the
skeleton is omitted entirely (D-13). No placeholder "No embed available" text is
rendered — the layout simply skips that block.

**Hydration safety:** `next/dynamic({ ssr: false })` is mandatory (STATE.md). No
exceptions. Planner must enforce this in code review.

### JSON-LD (REL-07)

Inline `<script type="application/ld+json">` rendered by the page route (D-22). No
separate API endpoint.

```json
{
  "@context": "https://schema.org",
  "@type": "MusicAlbum",
  "name": "{title}",
  "byArtist": [
    { "@type": "MusicGroup", "name": "{artistName1}" },
    { "@type": "MusicGroup", "name": "{artistName2}" }
  ],
  "datePublished": "{releaseDate YYYY-MM-DD}",
  "image": "https://marginalialabel.com/images/releases/{coverArtFilename}",
  "description": "{plain-text excerpt of description, truncated to 500 chars}",
  "url": "https://marginalialabel.com/releases/{slug}",
  "genre": ["{genre labels, human-readable}"]
}
```

- `byArtist` is always an array (even for single-artist releases) per Schema.org best
  practice for music catalogs.
- `genre` uses human-readable labels (`"Melodic House"`, not `"melodic-house"`).
- `image` and `url` must be absolute.
- `description` is the plain-text flattening of the Keystatic rich-text document
  (strip formatting), truncated to 500 chars. If empty, the field is omitted (schema
  validators tolerate missing `description`; they do not tolerate empty string).

## Token Additions

Phase 3 introduces **no new `@theme {}` tokens**. All design tokens were shipped in
Phase 2 and are sufficient for this phase. If the executor discovers a gap during
implementation (e.g. a specific border-radius token for the Laylo CTA that isn't yet
declared), they should flag it back for a UI-SPEC amendment rather than hardcode a
magic number.

**Hardcoded values allowed in Phase 3 without new tokens:**
- `166px` — SoundCloud iframe height (platform-standard, not a design decision)
- `rgba(31, 31, 33, 0.70)` — hover overlay (derived from `--color-bg` with opacity;
  alpha is the only new dimension)
- Aspect ratios `1/1` and `aspect-square` — Tailwind utility, no token needed
- Grid templates `grid-cols-3`, `grid-cols-4`, `grid-cols-5` — Tailwind utility

Any other hardcoded pixel, color, or size value is a contract violation and must be
routed through an `@theme {}` token.

## Accessibility Contract

- All platform links (primary and secondary rows) have descriptive `aria-label`
  (pattern: `{Platform} — {release title}`), not just the generic `{Platform}` used in
  Phase 2 footer.
- Cover art `<img>` elements use meaningful `alt` text (`{title} — {artists} cover
  artwork`), not `alt=""`.
- Hover overlay on catalog cards is **visual only** — title and artist are NOT in the
  accessibility tree of the card link. Screen readers get the full name via the
  `<a aria-label="{title} by {artists}">` on the card itself.
- The native `<details>` element for "More platforms" provides built-in keyboard and
  screen-reader support — no ARIA is added on top.
- Release detail page uses one `<h1>` (the release title). No multiple H1s.
- Color contrast: body text on `--color-bg` = 21:1 (WCAG AAA). Label text
  `--color-text-secondary` on `--color-bg` = 12:1 (AAA). Genre chip text checked per
  row — all pairings pass WCAG AA (4.5:1) minimum.
- Focus-visible outlines must not be removed on any element (inherited from Phase 2).
