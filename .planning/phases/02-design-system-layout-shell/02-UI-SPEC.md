# Phase 2 — UI Design Contract

> Visual and interaction contract for Phase 2: Design System & Layout Shell.
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.
>
> **Brand context:** Marginalia is a melodic house & techno label based in Barcelona.
> Dark-first, high-contrast, underground aesthetic. Release artwork is the primary
> visual element — the shell must recede and let the music lead.

## Spacing Scale

Declared values (all multiples of 4). Implemented as CSS custom properties in `globals.css`
under `@theme {}`.

| Token | CSS var | Value | Usage |
|-------|---------|-------|-------|
| xs | `--space-xs` | 4px | Icon gaps, inline tag padding |
| sm | `--space-sm` | 8px | Nav item internal padding, button padding-y |
| md | `--space-md` | 16px | Default element spacing, card padding |
| lg | `--space-lg` | 24px | Section padding, grid gap |
| xl | `--space-xl` | 32px | Layout column gaps |
| 2xl | `--space-2xl` | 48px | Major section breaks, footer padding-y |
| 3xl | `--space-3xl` | 64px | Page-level top/bottom padding |

Exceptions:
- Touch targets (mobile nav links, footer social icons): minimum 44px height — achieved
  via padding rather than a separate spacing token
- Nav horizontal padding on mobile: 20px (5-multiple — aligns with screen edge comfort)

## Typography

Font family for all UI text: `Nimbus Sans`, fallback `Helvetica Neue, Arial, sans-serif`.
Two weights only: Regular (400) and Bold (700). No italic in UI.

| Role | Size | Weight | Line Height | CSS var |
|------|------|--------|-------------|---------|
| Body | 16px | 400 (Regular) | 1.5 | `--text-body` |
| Label / Meta | 12px | 400 (Regular) | 1.4 | `--text-label` |
| Heading | 24px | 700 (Bold) | 1.2 | `--text-heading` |
| Display | 48px | 700 (Bold) | 1.1 | `--text-display` |

**Notes:**
- Body 16px is the minimum on mobile (DSYS-03 success criterion: "clean and legible at 16px on mobile")
- Display size scales to 64px on desktop (≥1024px) via a single responsive override
- Heading scales to 32px on desktop
- Label used for: nav subnav items, release catalog numbers, event dates, footer link groups
- Letter-spacing on Display and Heading: -0.02em (tight, bold editorial feel)
- All font sizes defined as `@theme {}` tokens in `globals.css` — never hardcoded in components

**Font loading implementation:**
```css
@font-face {
  font-family: 'Nimbus Sans';
  src: url('/fonts/NimbusSans-Regular.woff2') format('woff2');
  font-weight: 400;
  font-style: normal;
  font-display: swap;
}
@font-face {
  font-family: 'Nimbus Sans';
  src: url('/fonts/NimbusSans-Bold.woff2') format('woff2');
  font-weight: 700;
  font-style: normal;
  font-display: swap;
}
```

Source: `<brand_data>` — Nimbus Sans locked, self-hosted woff2, two weights only.

## Layout Shell Specification

Phase 2 delivers the global nav and footer used on every page (DSYS-05). Detailed here
because this is the entire UI deliverable of this phase.

### Breakpoints

| Name | Width | Behavior |
|------|-------|----------|
| mobile | 0–767px | Stack layout, hamburger nav |
| tablet | 768–1023px | Horizontal nav, condensed |
| desktop | 1024–1439px | Full nav with all items visible |
| wide | 1440px+ | Max-width container centered at 1280px |

Container max-width: 1280px, centered, 20px horizontal padding on mobile, 32px on tablet+.

### Global Navigation

**Structure:**
- Logo / wordmark (left) — links to `/`
- Primary nav links (right, horizontal on desktop): Home · About · Releases · Free Downloads · Merch · Podcasts · Showcases · Demo Submission · Subscribe · Press
- Mobile: hamburger toggle, nav links in a fullscreen overlay

**Behavior:**
- Nav background: `--color-surface` (`#2A2A2C`)
- Nav links: `--color-text-primary` at weight 400, 14px (Label size)
- Active link: `--color-accent-lime` bottom border, 2px
- Hover link: color shifts to `--color-surface-purple` (#8656FF)
- Logo: Nimbus Sans Bold or SVG wordmark — never the blackletter display font
- Nav height: 64px on desktop, 56px on mobile
- Position: sticky (top: 0), z-index: 50

**Mobile hamburger menu:**
- Icon: 3-line SVG (24×24px), `--color-text-primary`
- Open state: fullscreen overlay, `--color-bg` background, all nav links stacked at 24px
  (Heading size), weight 700
- Close: X icon in same position, tap anywhere outside closes
- Transition: 200ms ease-in-out opacity + translate-y

**Incubation / secondary nav:**
- Not in primary nav bar
- Linked from footer under "Incubation" label group:
  Management · Mix & Mastering · Production Classes · Mentoring
- Pages do not exist yet in Phase 2 — links are rendered as non-linked text with
  "(coming soon)" label, or href="#" with aria-disabled="true" until Phase 5+

### Footer

**Structure (3 columns on desktop, stacked on mobile):**

Column 1 — Brand:
- Logo or wordmark
- Short tagline (pulled from `siteConfig.tagline` singleton, fallback: "Barcelona · Melodic House & Techno")

Column 2 — Navigation:
- Quick links: Releases · Artists · Podcasts · Press · Showcases · About

Column 3 — Connect:
- Social icons row: Instagram · SoundCloud · Beatport · YouTube · TikTok · Facebook
  (all URLs from `siteConfig` singleton; icon not rendered if URL is empty)
- "Incubation" subgroup: Management · Mix & Mastering · Production Classes · Mentoring

**Footer bottom bar:**
- Copyright line: "© {year} Marginalia. All rights reserved."
- Background: `--color-bg` (#1F1F21), border-top: 1px solid rgba(255,255,255,0.08)
- Footer padding: 48px top, 32px bottom

**Social icons:**
- Inline SVG, 20×20px
- Color: `--color-text-secondary` (#D2D2DB)
- Hover: `--color-accent-lime` (#9EFF0A)
- Minimum touch target: 44×44px achieved with padding wrapper

## Interaction States

All interactive elements must declare all four states. No stateless interactive elements.

| State | Treatment |
|-------|-----------|
| Default | As specified in color contract above |
| Hover | Color shift per element spec above; transition 150ms ease |
| Focus (keyboard) | `outline: 2px solid #9EFF0A` (lime), `outline-offset: 3px` — no browser default |
| Active / Pressed | Violet (#580AFF) fill on buttons; 90% opacity on links |
| Disabled | 40% opacity, `cursor: not-allowed`, no hover effect |

---

## Responsive Contract

Phase 2 must pass DSYS-04: "no broken layouts, overflow, or content clipping from 375px to 1440px."

| Viewport | Nav | Footer | Container |
|----------|-----|--------|-----------|
| 375px (iPhone SE) | Hamburger menu | Single column stacked | 20px padding |
| 768px (iPad) | Horizontal links, may wrap to 2 lines | 2-column | 32px padding |
| 1024px (small desktop) | All primary links visible on one line | 3-column | 32px padding |
| 1440px | Same as 1024px, container max-width kicks in | 3-column | centered |

No horizontal scroll at any viewport. `overflow-x: hidden` on `body` as safety net.

## `globals.css` Token Implementation

The complete `@theme {}` block for this phase. Executor must write these into
`app/globals.css`, replacing the create-next-app defaults entirely.

```css
@import "tailwindcss";

/* ─── Nimbus Sans Self-Hosted Font ─────────────────────────── */
@font-face {
  font-family: 'Nimbus Sans';
  src: url('/fonts/NimbusSans-Regular.woff2') format('woff2');
  font-weight: 400;
  font-style: normal;
  font-display: swap;
}
@font-face {
  font-family: 'Nimbus Sans';
  src: url('/fonts/NimbusSans-Bold.woff2') format('woff2');
  font-weight: 700;
  font-style: normal;
  font-display: swap;
}

/* ─── Design Tokens ─────────────────────────────────────────── */
@theme {
  /* Colors — Core */
  --color-bg:              #1F1F21;
  --color-surface:         #2A2A2C;
  --color-accent-violet:   #580AFF;
  --color-accent-lime:     #9EFF0A;
  --color-surface-purple:  #8656FF;

  /* Colors — Text */
  --color-text-primary:    #FFFFFF;
  --color-text-secondary:  #D2D2DB;
  --color-text-muted:      #CAC9F9;

  /* Colors — Semantic */
  --color-destructive:     #ef6b8e;

  /* Colors — Tag palette (11 accent colors for genre chips) */
  --color-tag-pink:        #ef6b8e;
  --color-tag-orange:      #f29753;
  --color-tag-yellow:      #f9c432;
  --color-tag-olive:       #c0c020;
  --color-tag-mint:        #66cc99;
  --color-tag-green-light: #7ed35e;
  --color-tag-green-dark:  #599f56;
  --color-tag-lavender:    #b088d0;
  --color-tag-purple:      #bd63ee;
  --color-tag-blue:        #2086c0;
  --color-tag-sky:         #a9c2e7;

  /* Typography */
  --font-sans: 'Nimbus Sans', 'Helvetica Neue', Arial, sans-serif;

  --text-label:   0.75rem;   /* 12px */
  --text-body:    1rem;      /* 16px */
  --text-heading: 1.5rem;    /* 24px — scales to 2rem/32px on desktop */
  --text-display: 3rem;      /* 48px — scales to 4rem/64px on desktop */

  --leading-tight:  1.1;
  --leading-snug:   1.2;
  --leading-normal: 1.5;
  --leading-label:  1.4;

  --tracking-tight: -0.02em;

  /* Spacing */
  --space-xs:  0.25rem;   /* 4px */
  --space-sm:  0.5rem;    /* 8px */
  --space-md:  1rem;      /* 16px */
  --space-lg:  1.5rem;    /* 24px */
  --space-xl:  2rem;      /* 32px */
  --space-2xl: 3rem;      /* 48px */
  --space-3xl: 4rem;      /* 64px */

  /* Layout */
  --max-width-content: 1280px;
  --nav-height-desktop: 64px;
  --nav-height-mobile:  56px;
}

/* ─── Base ───────────────────────────────────────────────────── */
*, *::before, *::after {
  box-sizing: border-box;
}

html {
  background-color: var(--color-bg);
  color: var(--color-text-primary);
  font-family: var(--font-sans);
  font-size: 16px;
  -webkit-font-smoothing: antialiased;
  -moz-osx-font-smoothing: grayscale;
}

body {
  min-height: 100vh;
  overflow-x: hidden;
  background-color: var(--color-bg);
}

/* ─── Focus ──────────────────────────────────────────────────── */
:focus-visible {
  outline: 2px solid var(--color-accent-lime);
  outline-offset: 3px;
}
```
